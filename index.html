排序算法

一 . 前言

排序算法是数据处理中的经常执行的操作。高效率的排序应当尽可能减少比较次数和数据的移动次数。本文主要是探讨内排序，不涉及使用外存的外排序，主要是七种排序方法，包括冒泡，简单选择，直接插入，希尔，堆，归并，快速。

二. 七类算法

1. 冒泡排序

冒泡排序思路是：首先将序列当中的左右元素，依次比较，保证右边的元素始终大于左边的元素；对序列当中剩下的n-1个元素继续进行左右元素比较。对于长度为n的序列，一共需要执行n-1轮比较。

比较时会进行两轮的遍历，因此时间复杂度为O(n^2)。

希尔排序是针对直接插入排序的改进，它的算法思想：将待排序数组按照步长gap进行分组，然后将每组的元素利用直接插入排序的方法进行排序；每次将gap折半减小，循环上述操作；当gap=1时，利用直接插入，完成排序。

5. 堆排序

堆排序是针对直接选择排序的改进。首先是介绍堆这种数据结构。

堆可以表示为完全二叉树。特别重要的一点性质：任意的叶子节点小于（或大于）它所有的父节点。对此，又分为大顶堆和小顶堆，大顶堆要求节点的元素都要大于其孩子，小顶堆要求节点元素都小于其左右孩子，两者对左右孩子的大小关系不做任何要求。

利用堆排序，就是基于大顶堆或者小顶堆的一种排序方法。

6. 归并排序

归并排序是建立在归并操作上的一种有效的排序算法。该算法是采用分治法（Divide and Conquer）的一个非常典型的应用。通过先递归的分解数列，再合并数列就完成了归并排序。
归并排序的效率是比较高的，设数列长为N，将数列分开成小数列一共要logN步，每步都是一个合并有序数列的过程，时间复杂度可以记为O(N)，故一共为O(N*logN)。

具体来说，考虑下如何将二个有序数列合并。其做法是，只要从比较二个数列的第一个数，谁小就先取谁，取了后就在对应数列中删除这个数。然后再进行比较，如果有数列为空，那直接将另一个数列的数据依次取出即可。

7. 快速排序

快排是对于冒泡的改进和加强。通过一趟排序将待排序的记录分割成独立的两部分，其中一部分记录的关键字均比另一部分记录的关键字小，则可分别对于这两部分记录继续进行排序，以达到整个序列有序的目的。
快速排序是C.R.A.Hoare于1962年提出的一种划分交换排序。它采用了一种分治的策略，通常称其为分治法。
该方法的基本思想是：
1．先从数列中取出一个数作为基准数。
2．分区过程，将比这个数大的数全放到它的右边，小于或等于它的数全放到它的左边。
3．再对左右区间重复第二步，直到各区间只有一个数。

快排是不稳定的排序方法，时间复杂度为O(nlogn)。

优化思路：第一是基准数的选取，可以先选出3个数，取其平均数来作为基准，尽可能让这个基准数靠近中央。第二是递归方面，尽量使用尾递归。

四.路由的方法们

1.编程式导航

首先是编程式导航方法，router.push、 router.replace 和 router.go 这三个方法。想要导航到不同的 URL，则使用 router.push 方法。这个方法会向 history 栈添加一个新的记录，所以，当用户点击浏览器后退按钮时，则回到之前的 URL。当你点击 时，这个方法会在内部调用，所以说，点击 等同于调用 router.push(…)。
router.replace跟 router.push 很像，唯一的不同就是，它不会向 history 添加新记录，而是跟它的方法名一样 —— 替换掉当前的 history 记录。
router.go(n)的参数是一个整数，意思是在 history 记录中向前或者后退多少步。
它们三个跟 window.history.pushState、 window.history.replaceState 和 window.history.go好像， 实际上它们确实是效仿 window.history API 的，尤雨溪大大如是说。

五.几组概念的比较

1.各个路由模式的区别

vue-router 默认 hash 模式 —— 使用 URL 的 hash 来模拟一个完整的 URL，于是当 URL 改变时，页面不会重新加载。
如果不想要很丑的 hash，我们可以用路由的 history 模式，这种模式充分利用 history.pushState API 来完成 URL 跳转而无须重新加载页面。但是需要后台正确的配置。在 HTML5 history 模式下，router-link 会守卫点击事件，让浏览器不再重新加载页面。

hash: 使用 URL hash 值来作路由。支持所有浏览器，包括不支持 HTML5 History Api 的浏览器。 history: 依赖 HTML5 History API 和服务器配置。查看 HTML5 History 模式。
abstract: 支持所有 JavaScript 运行环境，如 Node.js 服务器端。如果发现没有浏览器的 API，路由会自动强制进入这个模式。

2.两种注入的属性

通过在 Vue 根实例的 router 配置传入 router 实例，下面这些属性成员会被注入到每个子组件。
一个是$router，它是一个router实例，包含路由匹配的模式和导航的相关方法。另一个是$route（路由信息对象）表示当前激活的路由的状态信息，包含了当前 URL 解析得到的信息，还有 URL 匹配到的 route records（路由记录）。像很多函数里面的from和to都是路由信息对象。链接较为详细地说明了路由信息对象的几种用处。另外router.currentRoute，是当前路由对应的路由信息对象，算是两者的关系吧。

3.重定向和别名

重定向的意思是，当用户访问 /a时，URL 将会被替换成 /b，然后匹配路由为 /b，那么别名又是什么呢？/a 的别名是 /b，意味着，当用户访问 /b 时，URL 会保持为 /b，但是路由匹配则为 /a，就像用户访问 /a 一样。

原理

轮播图的一个核心是图片的克隆。假设有五张图片，图片按照一定时间间隔自己进行轮播，当轮播到第五张图片时，如果直接轮播第一幅图片，会有图片一闪而过的现象。解决的思路是在第五张图片后面加上第一张图片的复制。换言之，如果以12345分别代表五幅图片，则轮播图要做的是变为123451，此举可谓精妙绝伦。不管从5顺序轮播到1，还是从1反向轮播到5，都有了一个第一张图片的复制作为缓冲，使得轮播不晃眼了。
