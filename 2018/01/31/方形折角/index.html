2.1 单位问题。

像素px是相对于显示器屏幕分辨率而言的，因此缩放可能会出现问题。严格来说，css的一像素不等于显示器的一像素。css 中的 px 不是物理像素. css px 和物理像素的对应关系, 与 viewport 的缩放有关。在windows里，默认的显示设置中，把文字定义为96dpi。这说明了：1px＝1/96英寸；而1pt＝1/72英寸，可以得出，在本设置中1px＝0.75pt，字体9pt＝12px。但是，用户可以修改文字定义的dpi大小，例如改成144dpi，这样，1px＝0.5pt，字体9pt＝18px。原先用12px来组成的一个文字，现在需要18px来组成，px多了，文字就“大”了，更易阅读了。

所以，px和pt的区别在于，px会随显示器的分辨率而改变，pt会随windows系统字体大小设置而改变。

em是相对于当前对象内文本的字体尺寸。如当前对行内文本的字体尺寸未被人为设置，则相对于浏览器的默认字体尺寸。em 准确来说，relative to the parent element。有一个极端的例子，体现em的问题

而rem，是CSS3新增的一个相对单位（root em，根em），使用rem为元素设定字体大小时，仍然是相对大小，但相对的只是HTML根元素。这个单位可谓集相对大小和绝对大小的优点于一身，通过它既可以做到只修改根元素就成比例地调整所有字体大小，又可以避免字体大小逐层复合的连锁反应。 准确说，always relative only to the root html element。
本文代码的0.5em，是相对于font-size大小的，浏览器一般默认是16px，font-size是24px，而border-radius是12px。

2.2 line-height

行高”顾名思意指一行文字的高度。具体来说是指两行文字间基线之间的距离。
div 的高度可以靠height撑开，也可以靠line-height。但是line-height不是靠文字撑起来的，而是靠在box模型中，有个line boxes，它可以包裹每行文字。一行文字一个line boxes。div的高度就是由一个一个line boxes的高度堆积而成。line boxes根据它包含的inline boxes里面谁的line-height值最高，就形成line boxes的高度。
一个不错的链接，值得一看。
你可以有5种方式来定义line-height。

父元素的line-height设为px，子元素直接继承；设为%或者em，那么首先根据父元素的font-size，
结合比例算出px，给子元素继承line-height，设为纯数字，子元素的行高根据自己的font-size乘上数字得到，相对自适应。px em %时候后代不会随着后代自己的font-size来改变行高，直接继承他爹的。line-height设置为normal，就是数值1.2，一般认为1.5更理想。
行高还有一个特性，叫做垂直居中性。line-height的最终表现是通过line boxes实现的，而无论line boxes所占据的高度是多少（无论比文字大还是比文字小），其占据的空间都是与文字内容公用水平中垂线的，可以用来给文字进行垂直居中。

2.6 box-shadow

按照MDN，box-shadow的取值为

inset
这个参数有的话，表示阴影在边框内，默认是没有，阴影在边框外
<offset-x> <offset-y>
这是头两个 <length>值，用来设置阴影偏移量。<offset-x> 设置水平偏移量，如果是负值则阴影位于元素左边。 <offset-y>设置垂直偏移量，如果是负值则阴影位于元素上面
<blur-radius>
这是第三个 <length> 值。值越大，模糊面积越大，阴影就越大越淡
<spread-radius>11
这是第四个 <length>值。取正值时，阴影扩大；取负值时，阴影.收缩。默认为0，此时阴影与元素同样大。
color：颜色值

2.7 before伪元素

In CSS, ::before creates a pseudo-element that is the first child of the selected element. It is often used to add cosmetic content to an element with the content property. It is inline by default.
还是觉得MDN的定义准确，::before在被选择的元素的首个孩子节点处创建一个伪元素，一般使用 content 属性来给元素添加内容，默认的before伪元素是行内元素。after以此类比，可得定义。
MDN上有一个使用before实现列表选项选中时，出现对勾符号的效果，对勾就是使用before添加的，对勾的形状居然是使用border的两个相邻边，然后rotate来模拟得到的！神奇！ 传送门 在此！
